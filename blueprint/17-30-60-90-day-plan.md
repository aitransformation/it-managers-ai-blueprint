# Section 17: 30 / 60 / 90 Day Implementation Plan

> **Journey Step:** All steps

---

## Overview

This 13-week plan covers the full implementation in weekly detail. Each week has a primary focus, specific deliverables, and a definition of done.

The plan assumes a single IT Manager with part-time support from one team member. All weeks assume a minimum of 8 hours dedicated to blueprint implementation alongside normal IT operations duties.

---

## Phase 1: Foundations (Weeks 1–4)

**Phase objective:** Infrastructure is built, policies are in place, data is classified, and the first agent capability is running in supervised mode.

---

### Week 1 — Leadership, Baseline, and Hardware

**Primary focus:** Secure mandate, establish measurements, order hardware.

**Deliverables:**
- Written senior leadership sponsorship document signed and filed
- Baseline measurements established for top 3 ROI target processes (time tracked for 5 working days)
- Hardware specification confirmed against recommended spec — order placed or existing server assessed
- IT Manager personal development plan drafted against the 5 competency standards
- AI literacy session scheduled for all staff (to be delivered in Week 3)

**Definition of done:**
- [ ] Leadership mandate document exists and is filed
- [ ] Baseline measurement spreadsheet created with first 5 days of data
- [ ] Hardware decision documented and actioned
- [ ] Development plan drafted

---

### Week 2 — Policy Foundation and Data Register

**Primary focus:** Get the policy and data classification foundations in place before any tool is deployed.

**Deliverables:**
- Acceptable Use Policy reviewed and AI Addendum drafted (for leadership approval in Week 4)
- AI Governance Policy — first draft
- Data Classification Policy — confirm or create the four-tier framework
- Begin data register — classify all data assets connected to planned AI use cases
- Outcome register started — first workshop with one operational function head

**Definition of done:**
- [ ] AUP AI Addendum drafted
- [ ] Data classification framework confirmed and documented
- [ ] Data register created with minimum 10 entries, classification confirmed by data owner
- [ ] Outcome register has minimum 3 outcomes defined with driver mapping

---

### Week 3 — Infrastructure Build

**Primary focus:** Build the physical and software foundation.

**Deliverables:**
- Ubuntu Server 22.04 LTS installed on dedicated hardware — all hardening steps applied
- NVIDIA drivers and CUDA installed — GPU detection verified
- Ollama installed — lightweight tier model pulled and tested (Phi-3 Mini or Llama 3 8B)
- Docker Compose application stack deployed — Apache, MariaDB, Gitea, Grafana, Prometheus
- HashiCorp Vault installed and initialised — root token secured
- AI literacy training (Tier 1) delivered to all staff

**Definition of done:**
- [ ] `ollama run phi3:mini` produces output with GPU layers confirmed
- [ ] All Docker services running and accessible on localhost
- [ ] Gitea accessible — first repository created (prompt-library)
- [ ] Vault unsealed and accessible — first secret stored
- [ ] Tier 1 training delivered and attendance logged

---

### Week 4 — OpenBrain and Model Routing

**Primary focus:** Stand up the central memory layer and configure model routing.

**Deliverables:**
- PostgreSQL installed with pgvector extension
- OpenBrain database schema deployed — all tables created
- OpenBrain MCP server running on localhost:8000
- nomic-embed-text embedding model pulled via Ollama — OpenBrain embedding tested
- OpenRouter account created — API key stored in Vault
- Agent routing configuration file created and committed to Gitea
- Mid-range model pulled (Llama 3 70B or Mistral Large) — tested against representative tasks
- AUP AI Addendum approved by leadership and distributed to all staff

**Definition of done:**
- [ ] `openbrain search "test"` returns results against test data
- [ ] OpenBrain MCP responding to `memory_store` and `memory_search` calls
- [ ] Agent routing config committed to Gitea with initial model assignments
- [ ] OpenRouter API key in Vault — test call confirmed
- [ ] AUP distributed and acknowledgement collected from all staff

---

## Phase 2: First Agents (Weeks 5–8)

**Phase objective:** Head Agent and four specialist agents deployed. First supervised autonomous tasks running. Governance framework complete.

---

### Week 5 — OpenClaw and Head Agent

**Primary focus:** Install the agent platform and deploy the Head Agent.

**Deliverables:**
- OpenClaw installed on bare metal — connected to Ollama and OpenBrain
- Head Agent configured — model routing, tool access, OpenBrain write permissions
- Agent task log tested — Head Agent writes correctly to `agent_tasks` table
- First 3 configuration tasks defined — input spec, verification criteria, rollback procedures written
- Runbooks written for all 3 configuration tasks
- Tier 2 training delivered to IT team

**Definition of done:**
- [ ] Head Agent responds to a test task request and logs to OpenBrain
- [ ] 3 task definitions committed to Gitea
- [ ] 3 runbooks written and stored in Gitea
- [ ] IT team Tier 2 training delivered — competency sign-off recorded

---

### Week 6 — Coding Agent and Script Library

**Primary focus:** Deploy the Coding Agent and establish the script library.

**Deliverables:**
- DeepSeek Coder 33B pulled via Ollama — tested on representative coding tasks
- Coding Agent configured in OpenClaw — connected to Gitea and OpenBrain
- Claude Code CLI installed on agent server — configured to use local Ollama endpoint as primary
- Script library seeded — all existing automation scripts committed to Gitea and registered in OpenBrain
- Local-script-first rule configured in Head Agent routing logic
- First agent-generated internal tool built — from requirements to Gitea commit
- QA Agent configured — test suite defined for Coding Agent outputs

**Definition of done:**
- [ ] Coding Agent generates a working script from a specified requirement
- [ ] Script committed to Gitea automatically by the agent
- [ ] Script registered in OpenBrain `script_library` with embedding
- [ ] `script_search` MCP call returns the new script when queried semantically
- [ ] QA Agent produces a structured review report for the generated script

---

### Week 7 — Monitoring Agent and Alert Routing

**Primary focus:** Connect monitoring inputs and deploy the Monitoring Agent.

**Deliverables:**
- Syslog export from firewall configured — logs flowing into rsyslog on agent server
- Prometheus node_exporter deployed on all monitored servers
- SNMP monitoring configured for network devices and UPS
- DMARC report inbox created — connected to agent parsing workflow
- Grafana AlertManager configured — webhook to OpenClaw
- Monitoring Agent configured — consuming alert webhook, writing to `monitoring_events` table
- Alert triage tested — 10 real or simulated alerts processed, triage summaries reviewed by IT Manager

**Definition of done:**
- [ ] Firewall logs appearing in rsyslog
- [ ] At least 3 Prometheus targets scraping successfully
- [ ] DMARC reports being parsed and written to OpenBrain
- [ ] Monitoring Agent producing triage summaries — IT Manager confirms quality
- [ ] All monitoring events appearing in `monitoring_events` SQL table

---

### Week 8 — Security Agent and Governance Agent

**Primary focus:** Complete the agent network for Phase 2.

**Deliverables:**
- Security Agent configured — CVE feed connected, security scan output parsing tested
- Security Agent produces first vulnerability prioritisation report — IT Manager reviews
- Governance Agent configured — AI Register and data register entries read and review dates monitored
- Governance Agent generates first governance summary — reviewed by IT Manager
- AI Register updated — all 6 deployed agents documented with full entries
- Monthly review meeting scheduled — recurring 90-minute calendar entry set

**Definition of done:**
- [ ] Security Agent produces a CVE prioritisation report for the current environment
- [ ] Governance Agent correctly identifies the next 3 governance review dates
- [ ] AI Register has entries for all 6 deployed agents
- [ ] Monthly review calendar entry confirmed for first occurrence

---

## Phase 3: Supervised Automation (Weeks 9–11)

**Phase objective:** Configuration task board running supervised. Service desk triage live. Reporting Agent deployed. First autonomous tasks approved.

---

### Week 9 — Configuration Task Board (Supervised)

**Primary focus:** Begin supervised execution of the first 5 configuration tasks.

**Definition of done:**
- [ ] All 5 tasks have run at least 3 times in supervised mode
- [ ] IT Manager has reviewed and approved all outputs
- [ ] No failures or unexpected outputs in the supervised runs
- [ ] Baseline measurement comparison documented

---

### Week 10 — Service Desk Triage (Shadow Mode)

**Primary focus:** Stand up service desk AI assistance in shadow mode.

**Definition of done:**
- [ ] 10 knowledge base entries in OpenBrain — verified resolutions for each
- [ ] Weekly IT health report generated and reviewed by IT Manager — quality confirmed
- [ ] Service desk shadow mode running — all tickets receiving agent recommendation
- [ ] IT Manager reviews 100% of agent recommendations in shadow mode

---

### Week 11 — Cloud Integrations and Skills Deepening

**Primary focus:** Cloud integrations live, IT Manager skills deepened.

**Definition of done:**
- [ ] M365 integration tested — agent can query user list via Graph API
- [ ] Google Workspace integration tested — agent can query user list via Admin SDK
- [ ] Both integrations in AI Register with permissions, data classifications, review dates
- [ ] Prompt library has minimum 15 entries — all with owners and review dates

---

### Week 13 — Quarterly Governance Review and Next Phase Planning

**Definition of done:**

## Day 90 — What good looks like

An IT Manager who has completed this plan will have:

**Infrastructure:**
- Dedicated agent server on local hardware, fully hardened, on a dedicated network segment
- Ollama running 3 model tiers locally (lightweight, mid-range, code-specialist)
- OpenClaw orchestrating 7 agents: Head, Coding, QA, Monitoring, Security, Governance, Reporting
- OpenBrain running PostgreSQL + pgvector with all SQL tables populated
- Full local application stack: LAMP, ISPConfig, Gitea, Grafana, Prometheus, Vault
- M365 and Google Workspace integrations live and documented
- Monitoring inputs from all major categories flowing into the monitoring stack

**Governance:**
- AI Register with entries for all 7 agents
- Data register with all critical data assets classified
- AUP AI Addendum approved and distributed
- AI Governance Policy, Data Classification Policy, and AI Data Processing Policy in place
- Complete SOP set — infrastructure, AI-specific, and security/compliance
- Prompt library with minimum 15 entries — all owned and reviewed
- Monthly governance review cadence established

**Operations:**
- Minimum 5 configuration tasks on the task board — at least 2 running autonomously
- Service desk triage in shadow or supervised mode for top 3 ticket types
- Weekly IT health report generating automatically
- Monitoring triage processing alerts from all major sources
- Security advisory producing weekly vulnerability prioritisation
- ROI measured and presented to leadership

**Skills and people:**
- Tier 1 training delivered to all staff
- Tier 2 training delivered to IT team — competency sign-off complete
- Internal champions identified and engaged
- IT Manager on a structured 2-hour/week development programme

This is a genuinely transformed IT operation. Not all of it will be perfect. Some supervised tasks will still be finding their feet. Some knowledge base entries will need updating. The monitoring triage will still occasionally surface false positives. That is normal.

The difference between day 1 and day 90 is not that everything works perfectly. It is that everything is measured, governed, improving, and delivering demonstrable value.
