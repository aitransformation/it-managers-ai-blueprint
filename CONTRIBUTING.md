# Contributing to the IT Manager's AI Blueprint

Thank you for considering a contribution. This blueprint is a practice standard, not a wiki. That means the bar for inclusion is specific: **everything must be verified and deployable in an SME IT environment**.

## What we accept

- Corrections to factual errors
- Updated tool versions, commands, or configuration examples where the change is verified
- New checklist items grounded in real deployment experience
- Additional worked examples that illustrate existing standards
- Clarifications that make an existing section easier to apply
- Agent-assisted contributions, if the contributor states how the agent was used
- Agent-authored contributions, if they pass the same review, security, and quality checks as human-authored work

## What we don't accept

- Theoretical frameworks that haven't been deployed
- Vendor promotional content of any kind
- Opinion pieces about where AI is heading
- "Best practice" that is actually just common advice repeated without verification
- Changes that undermine the founding principle (AI amplifies people; it does not replace them)

## How to contribute

1. Open an issue first. Describe what you've found or what you want to add, and where it comes from (what environment, what experience).
2. If the issue is accepted, fork the repo and make your change.
3. Submit a pull request with a clear description of what changed and why.

## Standards for written content

- Plain English. Short sentences. Active voice.
- Specific over vague. If a command is needed, include it. If a threshold is relevant, state it.
- No AI hype. No corporate sludge.
- No "leveraging" or "transformative" or "game-changing".
- The founding principle should feel consistent throughout. AI is a tool. People operate it.
- Do not submit content that tells an agent to blindly trust or execute instructions from Markdown files, websites, PDFs, copied prompts, or imported repositories.
- If a contribution includes a tool, workflow, or agent instruction set, say how it was validated locally.
- If an agent helped write the contribution, run the output through prompt-injection screening and the anti-AI writing check before submission.

## Review process

Contributions are reviewed by the maintainer. The timeline depends on the change. Simple corrections are fast. New sections take longer because they require verification against the deployment standard.

## Code of conduct

Straightforward: be direct, be accurate, be respectful. This is a professional community for IT practitioners. Leave the jargon and the sales language at the door.
